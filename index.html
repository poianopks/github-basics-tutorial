🐚 GitHub 왕초보 완전정복

회원가입부터 실습까지 한번에! 누구나 시작할 수 있는 GitHub 기초 가이드입니다.

히스토리 관리 협업 무료 플랜 제공 무료 배포(Pages) 오픈소스

📚 1. GitHub이란 무엇인가요?

GitHub은 코드를 저장하고, 협업하고, 배포할 수 있는 세계 최대의 개발자 플랫폼입니다.

🗂️

Git 저장소 (리포지토리)

Git은 코드의 변경이력을 추적하는 도구이고, GitHub는 그 Git 저장소를 인터넷에서 관리할 수 있게 해주는 플랫폼입니다.

🔄

버전 관리 (Version Control)

코드를 수정할 때마다 스냅샷식으로 저장해서 과거로 돌아갈 수 있습니다. 실수를 해도 안심!

🤝

팀 협업

여러 명이 동시에 같은 코드를 수정하고 합친 수 있습니다. 팡(Fork), 풀 리큐스트(PR)로 협업 가능.

🌐

웹젬리스도 지원

GitHub Pages를 이용해 HTML 스이트를 무료로 배포할 수 있습니다. 직접 도메인 URL도 제공!

🔍

오픈소스 허브

전 세계 개발자들의 코드가 모여있는 곳. 리눅스, Vue, React 등 유명 프로젝트들이 여기에 있습니다.

💼

포트폴리오

나만의 GitHub 계정은 개발자 포트폴리오! 코딩 실력을 절용원이나 클라이언트에게 보여줄 수 있습니다.

👤 2. 회원가입 따라하기

GitHub 계정을 만드는 방법을 단계별로 알아보걠습니다.

1

github.com 접속

브라우저에서 github.com으로 이동합니다. 우측 상단 Sign up 버튼을 클릭합니다.

2

이메일 주소 입력

유효한 이메일 주소를 입력합니다. 나중에 인증 메일을 수신하는데 실제 사용 중인 메일을 주세요.

팔스: Gmail, Naver 등 사용 가능. 한 이메일로 1개 계정만 만들 수 있습니다.
3

비밀번호 설정

대소문자+숫자+특수문자 포함 8자 이상의 비밀번호를 설정합니다.

⚠️ 분실 시 복구가 힘뒠니 비밀번호 관리자에 저장해 두세요!
4

유저네임 (Username) 선택

자신의 URL이 됩니다: github.com/유저네임. 나중에 바꾸기 힘뒠니 신중하게 선택하세요.

팔스: 영문 소문자+숫자+하이픈(-) 사용 가능. 한글 불가.
5

이메일 인증

가입 후 GitHub에서 인증 메일을 전송합니다. 메일함으로 가서 클릭하면 가입 완료!

6

2단계 인증 (2FA) 권장

Settings → Password and authentication → Two-factor authentication 설정. 보안을 크게 높일 수 있습니다.

⚙️ 3. GitHub 기본 사용법

리포지토리 생성부터 파일 업로드까지 기본 조작을 알아보걠습니다.

1

리포지토리(Repository) 만들기

GitHub 로그인 후 우측 상단 + 버튼 → New repository. 이름 입력 → Public/Private 선택 → Create repository.

2

파일 업로드 (Add file)

리포지토리 페이지에서 Add file → Upload files 또는 Create new file을 클릭해 파일을 추가할 수 있습니다.

3

Commit (코밋트) - 변경사항 저장

파일을 수정한 후 하단의 Commit changes 버튼을 클릭합니다. 코밋트 메시지는 어떤 내용을 바꾸었는지 설명하는 메모입니다.

feat: 로그인 기능 추가
fix: 로그인 버그 수정
docs: README 업데이트
4

Branch (브랜치) - 업무 분리

코드를 안전하게 개발하는 방식. main 브랜치에서 직접 작업하는 대신 새 브랜치를 만들어 작업합니다.

5

Pull Request (PR) - 코드 합치기

브랜치에서 작업한 내용을 main에 합치게 요청하는 기능. 팀 협업의 핵심!

6

Issues (이슈) - 작업 관리

버그 신고나 할 일 목록을 기록하는 제도. 라벨과 마일스톤으로 체계적으로 관리할 수 있습니다.

⭐ 4. GitHub을 사용하면 좋은점

왕초보부터 전문가까지 GitHub을 써야 하는 이유를 정리했습니다.

🛡️

코드 일일 백업

실수로 삭제하거나 실패해도 지난 버전으로 돌아갈 수 있습니다. 코드를 잘못 수정해도 겁내기 X!

🌍

어디서나 접근

인터넷이 되는 곳이라면 어디서든 코드를 확인하고 수정할 수 있습니다. PC, 팬드랙, 스마트폰 모두 사용 가능.

💰

무료 호스팅 (GitHub Pages)

추가 비용 없이 HTML/CSS 사이트를 배포할 수 있습니다. username.github.io 도메인이 무료로 제공됩니다.

📊

활동 이력 가시화

코밋트할 때마다 초록색 사각형이 씟히는 코드 일력(Contribution graph). 나의 노력을 한눈에!

🎓

개발자 취업 및 포트폴리오

많은 회사들이 쳨용시 GitHub 계정을 확인합니다. 코드 작성 습관을 보여주는 추서서!

🤖

CI/CD 자동화 (Actions)

코드를 push할 때마다 자동으로 테스트, 빌드, 배포하는 자동화 파이프라인을 구현할 수 있습니다.

👥 5. 누가 GitHub을 사용하나요?

GitHub은 개발자만 쓰는 것이 아닙니다. 다양한 사람들이 사용합니다.

💻

실력자 개발자

HTML, Python, Java 등 코드 프로젝트를 관리하고 취업 포트폴리오로 활용

👨‍👩‍👧

학생 / 코딩 입문자

코딩 연습문제를 저장하고 선생님과 코드 공유 및 코드 리뷰

📝

블로거 / 글쓰는 사람

Markdown 파일로 뼔로그를 쓰거나 문서를 GitHub Pages로 무료 배포

🏬

스타트업 / 회사

서비스 소스코드 관리, 팀 협업, CI/CD 자동화 파이프라인 활용

🌏

오픈소스 기여자

Linux, Vue.js, React 같은 오픈소스 프로젝트에 PR로 직접 기여

🎮

인디 게임 개발자

게임 소스 리소스 버전관리와 업데이트 히스토리 관리에 활용

🎙️

음악 프로듀서

음악 제작 스크립트, AI 음악 생성 코드를 GitHub에 저장하고 공유

🏷️

쓼몰운영 / 설정자

온라인 스토어 상품 데이터와 자동화 스크립트를 버전관리하며 운영

🚀 8. 실습 따라하기 — 첫 HTML 사이트 만들고 배포하기

GitHub Pages를 이용해 나만의 HTML 사이트를 단계별로 만들어 배포해 보겠습니다.

1

GitHub 로그인 후 리포지토리 생성

github.com 접속 → 우측 상단 + 버튼 → New repository 선택

Repository name: my-first-site
Visibility: Public
Add README: 체크 ✓
Create repository 클릭
2

index.html 파일 생성

Add file → Create new file 클릭 → 파일명에 index.html 입력

<!DOCTYPE html>
<html lang="ko">
<head><title>나의 첫 사이트</title></head>
<body>
  <h1>안녕하세요! GitHub Pages를 마듦습니다 🎉</h1>
</body>
</html>
3

Commit changes

하단 Commit changes 버튼 클릭 → 커밋트 메시지 입력 → Commit changes 확인

커밋트 메시지 예시: "feat: 쳋 HTML 파일 추가"
4

GitHub Pages 활성화

Settings 탭 클릭 → 왼쪽 메뉴에서 Pages 선택 → Source: main 브랜치 선택 → Save

⚠️ 배포는 약 1시간 안에 완료됩니다. 조금 기다리세요!
5

배포 URL 확인

Settings → Pages에서 배포된 URL을 확인합니다.

https://유저네임.github.io/my-first-site/
케주: 리포지토리 이름을 username.github.io로 만들면 단순 URL로 바로 접속 가능!
6

업데이트 및 커스터마이징

index.html을 수정하고 Commit하면 몰 분 안에 사이트가 자동 업데이트됩니다. CSS, JavaScript 파일도 추가해 보세요!
